# Stage 1 — Idea refinement

Generate `./planmysaas-blueprint/01-idea.md` with the structure below. This is the foundation every later stage reads from — get it specific.

## Output structure

```markdown
# <Project name>

> One-line summary

## Problem statement
2-3 sentences. What's broken today? Who feels the pain? Why hasn't it been solved already?

## Target audience
The primary buyer (one persona, named). Secondary audiences (max 2). For each: demographics, where they hang out, what they currently use, willingness to pay.

## Business model
Pick ONE primary: subscription / usage-based / marketplace / one-time / freemium. Justify in one sentence. Add price point hypothesis (range OK).

## Key features (top 5)
- Feature name — 1-line purpose. Mark which solve the core problem vs which are retention.

## Founder's rules
2-3 hard constraints from the founder (e.g. "must work offline", "no AI tokens spent on free tier", "Indian market only").

## Success criteria
What does month 6 look like if this works? Pick 1-2 measurable outcomes.

## Anti-goals
2-3 things this product is explicitly NOT. Helps scope.
```

## Quality rules

- **Name** must be concrete (not "X for Y"). If user gave only a category, propose 2 names + ask user to pick.
- **Problem** must be specific to the audience — generic pain ("planning is hard") is rejected.
- **Business model** must include a number (price or unit economics hypothesis).
- **Anti-goals** are mandatory — they prevent feature creep later.

## Tone

Plain English. Active voice. No marketing fluff. Read like a brief, not a pitch deck.
